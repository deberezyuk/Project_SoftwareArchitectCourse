Сценарий 1 - Оформление умного полиса в системе

1. Внешняя фронт-система отправляет синхронный HTTP-запрос (POST) с данными о клиенте, автомобиле и полисе в External API Gateway.
2. API Gateway направляет запрос в Сервис полис (Policy Service) – единую точку входа для всех операций с полисами.
3. Сервис полис выполняет следующие шаги в рамках одной транзакции (или саги):
4. Синхронно вызывает Сервис страхователь для создания/получения записи о клиенте. Возвращается policyholderId.
5. Синхронно вызывает Сервис авто для создания записи об автомобиле (связывая с policyholderId). Возвращается vehicleId.
6. Создаёт запись о полисе в своей БД, связывая с полученными идентификаторами.
7. После успешного сохранения Сервис полис публикует событие PolicyCreated в топик Kafka (например, policy-events).
8. Сервис профиля вождения (DrivingAnalytics) подписан на это событие. При получении он создаёт пустой профиль вождения для данного клиента/автомобиля со статусом "ожидает авторизации".
9. Сервис уведомлений также может подписаться на событие и отправить клиенту приветственное сообщение или инструкцию по авторизации через External API GW через Омниканальную платформу
10. Сервис полис возвращает синхронный ответ через API Gateway во внешнюю систему с подтверждением создания полиса и данными для дальнейшей авторизации.
